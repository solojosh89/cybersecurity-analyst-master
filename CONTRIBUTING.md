# Contributing

Thanks for considering a contribution to Cybersecurity Analyst Master.

## How to Contribute

1. **Fork** the repository and create a new branch from `main`.
2. Make your changes in the relevant folder:
   - `curriculum/` — learning path modules and syllabi
   - `labs/` — hands-on exercises (include clear objectives and setup steps)
   - `projects/` — larger applied projects
   - `notes/` — study notes and cheat sheets
   - `docs/` — general documentation
   - `website/` — website source
   - `assets/` — images and diagrams referenced elsewhere
3. Keep changes focused — one topic or fix per pull request.
4. Use clear, descriptive commit messages.
5. Open a pull request describing what changed and why.

## Content Guidelines

- Write in clear, beginner-friendly language where possible.
- Cite sources for any external material.
- Do not include real credentials, API keys, or sensitive data in labs or examples.
- Lab write-ups should include: objective, prerequisites, steps, and expected outcome.

## Reporting Issues

Use GitHub Issues to report broken links, outdated content, or suggest new topics.

## Code of Conduct

Be respectful and constructive. This is a learning-focused space — assume good faith.
