# Philosophy

When unsure about any decision in this project, check it against these. If it does not align, do not do it.

1. **The repo is the referee.** A session that produces no artifact produced nothing. Chat is a terminal; the repository is the operating system.
2. **Investigate before studying.** Problem first. Flounder. Then theory, when you need it and it can stick.
3. **Evidence beats opinion.** "Probably" is not a verdict. Show the path, the signer, the log line, the packet.
4. **Prove, don't assert.** A module is finished only when the answer to "can you prove you know this?" is yes, with artifacts.
5. **Understanding before tools.** A tool you cannot explain is a magic box, and magic boxes fail you mid-incident.
6. **Every lesson leaves evidence.** Eight artifacts, every module: lesson, lab, investigation, notes, quiz, portfolio, reflection, meaningful commit.
7. **Commits tell the growth story.** No filler commits. The history should read like a career forming.
8. **Content before platform.** Lessons have value without a website. A website has none without lessons. The site is a viewer, built in one weekend, later.
9. **Consistency beats intensity.** Three real hours a week that happen beat a twenty-hour plan that collapses.
10. **Build systems, not shortcuts.** Anything worth doing twice gets documented so the next learner can do it too.
11. **Every investigation claim follows Observation → Evidence → Reasoning → Conclusion → Confidence.** Not "this looks bad," but: what you saw, what proves it, why that matters, what you conclude, and how sure you are. This is the shape real incident responders write in, and it forces reasoning to be checkable instead of vibes.
